# AI 助手与主干同步策略

## 设计原则

为了降低后续与 Memos 主干合并时的冲突，本功能遵循以下原则：

1. 主链路只保留最小挂点。
2. 复杂逻辑放到独立目录。
3. 默认关闭，不影响现有核心功能。
4. 不修改现有 Memo、评论、用户的核心数据模型语义。

## 当前高风险冲突点

1. `server/router/api/v1/memo_service.go`
2. `server/router/api/v1/v1.go`
3. `server/server.go`
4. `web/src/components/Settings/AISection.tsx`

## 推荐同步方式

1. 将 AI 助手改动拆成独立补丁栈。
2. 每次同步主干后优先检查以上高风险文件。
3. 保持 `internal/aibot/` 尽量不依赖主干内部细节。
4. 当上游提供正式扩展点时，优先迁移到上游扩展点。

## 后续正式化路线

当前实现使用 JSON 配置文件，是为了规避当前环境中 proto 代码生成受阻的问题。

当前触发配置采用规则组列表，而不是复杂过滤表达式，以便降低 UI 复杂度并减少与主干过滤能力变更的耦合。

后续建议按以下顺序正式化：

1. 将配置迁移到 `InstanceSetting`。
2. 将内存队列迁移到持久化任务表。
3. 将外部适配器凭据接入正式敏感配置存储。
4. 将评论生成切换到正式 LLM 聊天接口。

## 回退策略

如果 AI 助手出现异常，可通过以下方式快速回退：

1. 在设置页关闭 `enabled`。
2. 删除 `data/config/ai-assistant.json`。
3. 移除 `memo_service.go` 中的入队调用。

这样可以在不影响 Memo 核心能力的情况下快速停用本功能。
